说明：

  • 字段类型基本都是 SQLite 的 INTEGER（含 64-bit 时间戳/ID）或 TEXT。表中标 enum 的整型字段是 CUPTI 枚举（取值见对应行）。
  • 时间戳单位 = 纳秒；以 nsys session 起点为零（对应 TARGET_INFO_SESSION_START_TIME）。
  • 跨 host↔device 关联的标准 join 键是 correlationId：RUNTIME / DRIVER 的一行（CPU launch）= KERNEL / MEMCPY / MEMSET / SYNCHRONIZATION 的一行（GPU 执行）。
  • 所有 nameId / shortName / demangledName 等 *Id 字段都是 StringIds(id) 的外键，需要 join 才能拿到字符串。
  • 不同 nsys 版本里某些列是 _v2 / _v3 版本（比如 localMemoryTotal_v2），含义不变，只是 schema 演进导致的别名。
